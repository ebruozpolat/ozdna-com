API Reference

The ozDNA REST API exposes the production AI gateway for B2B integrations. JSON request/response, Bearer token authentication, and OpenAPI 3.1 contract. Client libraries: SDK overview (Python, Node.js, Go, Java).

Introduction

The API provides AI text detection via /v1/detect — segment-level analysis, vertical mode calibration, and multilingual support. ozDNA does not offer text humanization or detection-evasion tooling.

The same production engine runs on tezmakale.com; the B2B API layer opens as backend infrastructure matures. See also: ozDNA production partners · TezMakale case study.

Authentication

All requests require an Authorization header with a Bearer token:

Authorization: Bearer ozdna_sk_live_...

API keys are created per project in the dashboard. Keys are prefixed with ozdna_sk_test_ (sandbox) or ozdna_sk_live_ (production).

HeaderValueRequired
AuthorizationBearer ozdna_sk_...yes
Content-Typeapplication/jsonyes
X-Request-IdUUID for idempotency tracingoptional

Never expose API keys in client-side code. Route requests through your backend or use the ozDNA gateway proxy pattern.

Quick Start

Get your first API response in three steps:

  1. Request access — Join the waitlist to receive sandbox credentials.
  2. Create an API key — Generate a test key in the dashboard (ozdna_sk_test_...).
  3. Send a detect request — POST text to /v1/detect with your Bearer token.
# 1. Set your API key
export OZDNA_API_KEY="ozdna_sk_test_..."

# 2. Detect AI-generated text
curl -X POST https://api.sandbox.ozdna.com/v1/detect \
  -H "Authorization: Bearer $OZDNA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text": "Sample academic text...", "mode": "academic", "language": "en"}'

Node.js SDK Example

// npm install @ozdna/sdk  (coming soon — use fetch below for now)
const OZDNA_API_KEY = process.env.OZDNA_API_KEY;

async function detect(text) {
  const res = await fetch('https://api.ozdna.com/v1/detect', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${OZDNA_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      text,
      mode: 'academic',
      language: 'en',
    }),
  });

  if (!res.ok) {
    const err = await res.json();
    throw new Error(`ozDNA ${res.status}: ${err.message}`);
  }

  return res.json();
}

detect('Sample text to analyze...')
  .then(result => console.log(result.ai_probability))
  .catch(console.error);

Python SDK Example

# pip install ozdna  (coming soon — use requests below for now)
import os
import requests

OZDNA_API_KEY = os.environ["OZDNA_API_KEY"]
BASE_URL = "https://api.ozdna.com"

def detect(text: str, mode: str = "academic", language: str = "en") -> dict:
    response = requests.post(
        f"{BASE_URL}/v1/detect",
        headers={
            "Authorization": f"Bearer {OZDNA_API_KEY}",
            "Content-Type": "application/json",
        },
        json={"text": text, "mode": mode, "language": language},
        timeout=30,
    )
    response.raise_for_status()
    return response.json()

if __name__ == "__main__":
    result = detect("Sample text to analyze...")
    print(f"AI probability: {result['ai_probability']}")

POST /v1/batch

POST /v1/batch

Bulk detect operations. Available on Growth and Enterprise plans. Processing is async — results delivered via webhook_url or polled at GET /v1/batch/{id}.

LLM Gateway

The production AI gateway exposes OpenAI-compatible inference routes through a single entry point. Every request passes authentication, quota checks, rate limits, model routing, and cost attribution. Response headers include X-OzDNA-Model, X-OzDNA-Cost-USD, and X-OzDNA-Cache: HIT on cached embeddings.

FeatureBehavior
AuthenticationBearer API key (ozdna_sk_test_ / ozdna_sk_live_)
QuotasMonthly quota enforced per key before upstream call
Rate limitsPer-key bucket; 429 with Retry-After
RetriesUpstream 429/502/503/504 — up to 3 attempts with backoff
CachingEmbeddings cached 60s per org + input hash
LoggingStructured logs + audit trail on every request

Optional body field mode (academic, legal, financial, general) steers model routing for chat and completion.

Admin REST API

Programmatic org management for automation and CI. Routes require a Bearer key scoped to the same organization. Keys can list and create resources within their org; the full secret is returned only once on key creation.

MethodPathDescription
GET/v1/orgs/:orgIdOrganization profile

POST/v1/orgs/:orgId/workspacesCreate workspace
GET/v1/workspaces/:workspaceId/projectsList projects
POST/v1/workspaces/:workspaceId/projectsCreate project
GET/v1/projects/:projectId/keysList keys (prefix only)
POST/v1/projects/:projectId/keysCreate key — secret shown once
DELETE/v1/projects/:projectId/keys/:keyIdRevoke key

Rate Limits

Rate limits are enforced per API key at the gateway layer. Exceeded limits return 429 Too Many Requests with a Retry-After header.

PlanRequests/minConcurrentMonthly quota
Developer (sandbox)1021,000 calls
Startup6010Custom
Growth30050Custom
EnterpriseCustomCustomUnlimited

Implement exponential backoff on 429 responses. Use X-RateLimit-Remaining and X-RateLimit-Reset headers to throttle proactively.

Error Codes

All errors return JSON with error, message, and request_id fields.

400Bad Request — missing text, invalid mode, or payload exceeds size limit
401Unauthorized — invalid, expired, or missing API key
402Payment Required — plan quota exceeded; upgrade or contact sales
403Forbidden — key lacks permission for this endpoint or vertical mode
404Not Found — invalid endpoint or batch job ID
429Rate Limit — check Retry-After header before retrying
500Internal Error — retry with exponential backoff; contact support if persistent
503Service Unavailable — temporary outage; retry with backoff
{
  "error": "rate_limit_exceeded",
  "message": "Rate limit exceeded. Retry after 30 seconds.",
  "request_id": "req_abc123"
}

Webhooks

For async requests with a webhook_url, a POST callback is sent on completion. Payload matches the endpoint response with additional job_id and event fields.

Verify signatures via the X-OzDNA-Signature header (HMAC-SHA256).
