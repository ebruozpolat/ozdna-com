API Referansı

ozDNA REST API, production AI gateway'ini B2B entegrasyonlara açar. JSON istek/yanıt, Bearer token kimlik doğrulama ve OpenAPI 3.1 sözleşmesi. İstemci kütüphaneleri: SDK genel bakış (Python, Node.js, Go, Java).

Giriş

API, AI metin dedeksiyonu sunar — /v1/detect üzerinden segment düzeyinde analiz, dikey mod kalibrasyonu ve çok dilli destek. ozDNA metin humanizasyonu veya dedeksiyon atlatma araçları sunmaz.

Aynı production motoru tezmakale.com üzerinde çalışır; B2B API katmanı backend olgunlaştıkça açılmaktadır. Ayrıca: ozDNA production partner'ları · TezMakale vaka çalışması.

Kimlik doğrulama

Tüm isteklerde Bearer token ile Authorization header'ı zorunludur:

Authorization: Bearer ozdna_sk_live_...

API anahtarları dashboard üzerinden proje kapsamında oluşturulur. Anahtarlar ozdna_sk_test_ (sandbox) veya ozdna_sk_live_ (production) öneki ile gelir.

Hızlı başlangıç

İlk API yanıtınızı üç adımda alın:

  1. Erişim isteyin — Sandbox kimlik bilgileri için waitlist'e katılın.
  2. API anahtarı oluşturun — Dashboard'da test anahtarı üretin (ozdna_sk_test_...).
  3. Detect isteği gönderin — Bearer token ile metni /v1/detect endpoint'ine POST edin.
# 1. API anahtarınızı ayarlayın
export OZDNA_API_KEY="ozdna_sk_test_..."

# 2. AI üretimi metni tespit edin
curl -X POST https://api.sandbox.ozdna.com/v1/detect \
  -H "Authorization: Bearer $OZDNA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"text": "Örnek akademik metin...", "mode": "academic", "language": "tr"}'

Node.js SDK örneği

// npm install @ozdna/sdk  (yakında — şimdilik fetch kullanın)
const OZDNA_API_KEY = process.env.OZDNA_API_KEY;

async function detect(text) {
  const res = await fetch('https://api.ozdna.com/v1/detect', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${OZDNA_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      text,
      mode: 'academic',
      language: 'tr',
    }),
  });

  if (!res.ok) {
    const err = await res.json();
    throw new Error(`ozDNA ${res.status}: ${err.message}`);
  }

  return res.json();
}

detect('Analiz edilecek örnek metin...')
  .then(result => console.log(result.ai_probability))
  .catch(console.error);

Python SDK örneği

# pip install ozdna  (yakında — şimdilik requests kullanın)
import os
import requests

OZDNA_API_KEY = os.environ["OZDNA_API_KEY"]
BASE_URL = "https://api.ozdna.com"

def detect(text: str, mode: str = "academic", language: str = "tr") -> dict:
    response = requests.post(
        f"{BASE_URL}/v1/detect",
        headers={
            "Authorization": f"Bearer {OZDNA_API_KEY}",
            "Content-Type": "application/json",
        },
        json={"text": text, "mode": mode, "language": language},
        timeout=30,
    )
    response.raise_for_status()
    return response.json()

if __name__ == "__main__":
    result = detect("Analiz edilecek örnek metin...")
    print(f"AI olasılığı: {result['ai_probability']}")

LLM Gateway

Production AI gateway, OpenAI uyumlu inference route'larını tek giriş noktasından sunar. Her istek kimlik doğrulama, kota kontrolü, rate limit, model yönlendirme ve maliyet atamasından geçer. Yanıt header'ları: X-OzDNA-Model, X-OzDNA-Cost-USD, önbellekten gelen embeddings için X-OzDNA-Cache: HIT.

ÖzellikDavranış
Kimlik doğrulamaBearer API anahtarı (ozdna_sk_test_ / ozdna_sk_live_)
KotalarUpstream çağrıdan önce anahtar başına aylık kota
Rate limitAnahtar başına bucket; 429 + Retry-After
Yeniden denemeUpstream 429/502/503/504 — en fazla 3 deneme, backoff ile
ÖnbellekEmbeddings org + input hash başına 60 sn
LoglamaYapılandırılmış log + her istekte audit trail

İsteğe bağlı mode alanı (academic, legal, financial, general) chat ve completion için model yönlendirmesini belirler.

Rate limit

Rate limit'ler gateway katmanında API anahtarı başına uygulanır. Limit aşımında 429 Too Many Requests ve Retry-After header'ı döner.

Planİstek/dkEşzamanlıAylık kota
Developer (sandbox)1021.000 çağrı
Startup6010Özel
Growth30050Özel
EnterpriseÖzelÖzelSınırsız

429 yanıtlarında exponential backoff uygulayın. Proaktif throttle için X-RateLimit-Remaining ve X-RateLimit-Reset header'larını kullanın.

Hata kodları

Tüm hatalar error, message ve request_id alanları içeren JSON döner.

400Geçersiz istek — eksik text, geçersiz mode veya boyut limiti aşımı
401Yetkisiz — geçersiz, süresi dolmuş veya eksik API anahtarı
402Ödeme gerekli — plan kotası aşıldı; yükseltin veya satış ile iletişime geçin
403Yasak — anahtar bu endpoint veya dikey mod için yetkili değil
404Bulunamadı — geçersiz endpoint veya batch job ID
429Rate limit — yeniden denemeden önce Retry-After header'ına bakın
500İç hata — exponential backoff ile yeniden deneyin; devam ederse destek
503Servis kullanılamıyor — geçici kesinti; backoff ile yeniden deneyin

Webhook'lar

webhook_url ile asenkron isteklerde işlem tamamlandığında POST callback gönderilir. Payload, endpoint yanıtı ile aynı yapıdadır; ek olarak job_id ve event alanları bulunur.

İmza doğrulama: X-OzDNA-Signature header'ı (HMAC-SHA256).
