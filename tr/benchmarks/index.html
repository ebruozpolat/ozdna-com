ozDNA vs
doğrudan sağlayıcı

Birleşik bir gateway'in uygulama kodunda OpenAI (veya başka bir sağlayıcı) doğrudan bağlanmasıyla gecikme, maliyet kontrolü ve operasyonel derinlik açısından karşılaştırması.

Örnek senaryo

academic modda POST /v1/detect çalıştıran varsayımsal B2B SaaS — ayda ~50.000 istek, karışık EN/TR metin, tek bölge dağıtımı. Yalnızca aşağıdaki karşılaştırmayı yapılandırmak için kullanılır.

  • Doğrudan sağlayıcı: Uygulama → OpenAI API (tek model, özel yeniden deneme mantığı, stdout günlükleri)
  • ozDNA: Uygulama → ozDNA Gateway → Model Router → Prompt Kayıt defteri → sağlayıcı

ozDNA'nın değer kattığı alanlar

Cost control

Direct integration optimizes for time-to-first-call. ozDNA optimizes for margin at scale — routing, cache, and per-workflow attribution.

Çalışıyor depth

Retries, observability, and prompt versioning are product features — not weekend engineering projects bolted onto a provider SDK.

Gecikme ödünleşimi

A gateway adds a hop. Illustrative example shows ~30 ms p50 delta — acceptable when routing and cache savings matter more than raw minimum latency.
