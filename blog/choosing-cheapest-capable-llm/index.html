Choosing the Cheapest Capable LLM for Each Task

The cheapest LLM for production is not the lowest price per token globally. It is the lowest-cost model that clears the quality bar for this step in this vertical workflow.

Teams ask for the cheapest LLM for production as if one model wins. In real products, you run a portfolio: tiny models for routing and extraction, mid-tier for drafting, premium only where errors are expensive.

Define the task, not the product

Break workflows into steps with explicit outputs: classify intent, extract entities, retrieve, synthesize, validate format. Each step gets its own capability requirement — legal citation vs. emoji tone check are not the same bar.

Run capability tiers with evals

  1. Collect 50–200 production-shaped inputs per step (redacted).
  2. Score outputs with automated checks + periodic human review.
  3. Try models from cheapest tier upward until pass rate meets threshold.
  4. Lock routing policy; re-run quarterly or on model/provider changes.

When premium is non-negotiable

Pay for flagship models when: errors have regulatory or financial liability, long-context reasoning is essential with no retrieval substitute, or user trust collapses after one bad answer. Everything else is a routing optimization candidate.

Avoid common mistakes

  • Using public MMLU scores for your compliance PDFs.
  • Downgrading generation before fixing retrieval.
  • Ignoring output token length — verbose cheap models can cost more than concise premium ones.
